This command generates non-existent classes such as factories and interceptors for plug-ins. It also generates the dependency injection configuration for the object manager. Generated code and the dependency injection configuration is written to the <your Magento install dir>/var/generation and <your Magento install dir>/var/di directories, respectively.

If <your Magento install dir>/var/di/definitions.php exists, it is used by Magento\Framework\ObjectManager\Definition\Compiled; otherwise, the much slower Magento\Framework\ObjectManager\Definition\Runtime is used. (This class uses reflection based the definition builder.)

Runtime definition compilation is fine in a development environment but you should use the compiler in a production environment.
