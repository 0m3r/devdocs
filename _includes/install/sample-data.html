ADD: Sample data as module, two ways to install

Introduction to Magento sample data

Magento sample data provides a storefront based on the Luma theme outfitted with products, categories, customer registration, and so on. It functions just like a Magento storefront and you can manipulate prices, inventory, and promotional pricing rules using the Magento Admin.

Installing sample data is optional; be aware that if you do install it, there is no way to uninstall only the sample data. You must uninstall and reinstall the Magento software.

This section discusses how to enable you to install the optional Magento sample data. To enable sample data, you must update composer.json in the Magento root installation directory to provide the location of the sample data package. After that you can either run the Magento software installer or you can run a script to install the sample data.

Sample data is versioned like Magento code. Before you begin, you can either:

  • Find the exact version you want at packages.magento.com.
  • Install the latest version using Composer next significant release syntax.

    • Get started

    To enable sample data, you must:

    1. Decide the version of magento/sample-data you want as discussed in Introduction to Magento sample data.
    2. Run the composer config and composer require commands to update composer.json.

    Update composer.json

    To update composer.json:

    1. Log in to your Magento server as the Magento file system owner or as a user with root privileges.
    2. Change to your Magento installation directory.
    3. Make a backup copy of composer.json. 
      cp composer.json composer.json.bak
    4. Open composer.json in a text editor.
    5. In the first section, add "minimum-stability": "beta", before license. A snippet follows: 
      "name": "magento/project-community-edition",
    "description": "Magento project (Community Edition)",
    "type": "project",
    "minimum-stability": "beta",
    "version": "1.0.0-beta",
    "license": [
        "OSL-3.0",
        "AFL-3.0"
    ],
    6. Save your changes to composer.json and exit the text editor.
    7. Enter the following command to reference Magento packages in composer.json:
      composer config repositories.magento composer http://packages.magento.com
    8. Enter the following command to require the current version of the sample data package:
      composer require magento/sample-data:<version>
      where <version> is either an exact version or semantic version syntax.
      Exact version example:
      composer require magento/sample-data:1.0.0-beta
      Next significant release example:
      composer require magento/sample-data:~1.0.0-beta
    9. Wait while dependencies are installed.
